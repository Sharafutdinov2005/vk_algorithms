# Объяснение однопроходностим

Рассмотрим код функции `find_list_mid`

```python
    fast = head
    slow = head

    while fast is not None and fast.next is not None:
        slow = slow.next
        fast = fast.next.next

    return slow
```

Путь на вход дан список длиной $2n + 1$. Когда быстрый указатель дойдёт с $0$-го до $2n$-го элемента, сделав $2n$ шагов, медленный сделает $n$ шагов, и окажется на середине.

Если же на вохд дан список длиной $2n$, то быстрый указатель с $0$-го до $2n - 2$-го (он ходит только по чётным) дойдёт за $n-1$ шаг, и медленный остановится в окрестности середины.

В общем, идея в том, что и медленный, и быстрый делают равное кол-во шагов.
