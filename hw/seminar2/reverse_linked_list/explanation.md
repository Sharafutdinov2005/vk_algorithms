# Изменение указателей в цикле

Рассмотрим код функции `reverse_linked_list`:

```python
if head.next is None:
        return head

    prev = None
    curr = head

    while curr is not None:
        next = curr.next
        curr.next = prev
        prev = curr
        curr = next

    return prev
```

Посмотрим на примере, как меняются указатели. Пусть на вход дан список

```
1 -> 2 -> 3 -> 4 -> None
```

Перед началом цикла `prev = None`, `curr = 1`.

В цикле мы сохраняем указатель на `2`, чтобы не потерять его, и переписываем указатель `1` на `None`. Теперь `prev` указывает на `1`, а `curr` - на `2`. В результате имеем:
```
2(curr) -> 3 -> 4 -> None

1(prev) -> None
```

Повторяем эту же операцию, получим:

```
3(curr) -> 4 -> None

2(prev) -> 1 -> None
```

И так далее.

Таким образом, сохраняя ссылку на предыдущий и следующий элементы, мы перезаписываем указатель текущего на прошлый, и не теряем следующие.

# Вывод временной сложности алгоритма

Рассмотрим код функции `reverse_linked_list`:

```python
if head.next is None:
        return head

    prev = None
    curr = head

    while curr is not None:
        next = curr.next
        curr.next = prev
        prev = curr
        curr = next

    return prev
```

Вычислим количество операций, выполняемых фукцией:
- до цикла функция выполняет постоянное количество операций:
$$
C_1
$$
- В цикле `curr` движутся от начала и конца списка по всем нодам, производя N операций:
$$
N * C_2 + C_1
$$

Таким образом итоговая асимптотика функции при больших N:
$$
O(N)
$$
