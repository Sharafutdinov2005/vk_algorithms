# Временная сложность

Рассмотрим код функции `merge_lists`:

```python
    result = OneLinkedList()

    while (len(list1) > 0) or (len(list2) > 0):
        if (
            len(list1) <= 0 or
            len(list2) > 0 and list2.head_value < list2.head_value
        ):
            result.push_back(list2.pop_front())
        else:
            result.push_back(list1.pop_front())

    return result
```

Вычислим количество операций, выполняемых фукцией:
- до цикла функция выполняет постоянное количество операций, создавая новый объект односвязного списка:
$$
C_1
$$
- В цикле два односвязных списка длиннами N и M последовательно удаляют свои ноды, производя N + M операций (за итерацию удаляется только одна). За $C_2$ обозначим усреднённое время выполнения условных операторов:
$$
(N + M) * C_2 + C_1
$$

Таким образом итоговая асимптотика функции при больших N:
$$
O(N + M)
$$

# Затраты памяти

В коде при удалении из исходных списков не копируются объекты нод, а напрямую перепривязываются к концу нового списка. Остальные операции требуют O(1) по памяти, и не влияют на пространственную сложность.
